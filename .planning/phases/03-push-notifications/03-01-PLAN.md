<objective>
Consolidate push token registration so every app launch registers the device token with the backend, and that token is the single source of truth for all push delivery.

Purpose: Two token-storage paths currently exist — `PUT /api/users/me/push-token` (users.py) stores token on `db.users`, and `POST /api/alerts/register` (alerts.py) stores token on `db.alert_preferences`. All push-sending code reads from `db.alert_preferences`. The users.py path is dead weight that creates confusion. We consolidate to `POST /api/alerts/register` as the canonical path.

Output: Single token registration flow, idempotent on every launch, verified working end to end.
</objective>
