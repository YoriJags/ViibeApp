Step 1 — Create c:/VIIBE/VibeApp/frontend/app.config.js.

This file replaces the static plugin config for @rnmapbox/maps. Expo will prefer app.config.js over app.json for plugin resolution.

```js
// app.config.js
// Dynamic config — injects Mapbox download token from EAS Secrets at build time.
// NEVER hardcode the token here. Store it in EAS Secrets as MAPBOX_DOWNLOAD_TOKEN.

const mapboxToken = process.env.MAPBOX_DOWNLOAD_TOKEN;

if (!mapboxToken) {
  console.warn(
    '[app.config.js] WARNING: MAPBOX_DOWNLOAD_TOKEN is not set. ' +
    'Mapbox SDK will not initialize. For local dev, set it in your shell or a .env.local file. ' +
    'For EAS builds, add it via: eas secret:create --scope project --name MAPBOX_DOWNLOAD_TOKEN'
  );
}

module.exports = ({ config }) => ({
  ...config,
  plugins: [
    ...(config.plugins || []).filter(
      (p) => !(Array.isArray(p) && p[0] === '@rnmapbox/maps')
    ),
    [
      '@rnmapbox/maps',
      {
        RNMapboxMapsDownloadToken: mapboxToken || '',
      },
    ],
  ],
});
```

Explanation of the filter: app.json still declares the @rnmapbox/maps plugin entry (needed for type resolution), but app.config.js removes it and re-adds it with the runtime token. This prevents double-registration.
