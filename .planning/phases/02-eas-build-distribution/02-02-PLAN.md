  <action>
Working directory for all eas commands: c:/VIIBE/VibeApp/frontend

Prerequisites (confirm before running build):
1. Plan 02-01 is complete — MAPBOX_DOWNLOAD_TOKEN secret is in EAS Secrets
2. Apple Developer account credentials are configured in EAS (`eas credentials`)
3. Bundle identifier com.viibe.app is registered in App Store Connect

Step 1 — Ensure eas CLI is authenticated:
```bash
eas whoami
```
If not logged in: `eas login`

Step 2 — Configure iOS credentials (first time only):
```bash
cd c:/VIIBE/VibeApp/frontend
eas credentials --platform ios
```
Select "Build Credentials" > let EAS manage certificates and provisioning profiles automatically.

Step 3 — Trigger the production iOS build:
```bash
cd c:/VIIBE/VibeApp/frontend
eas build --platform ios --profile production --non-interactive
```
This will:
- Upload source code to EAS
- Inject MAPBOX_DOWNLOAD_TOKEN from EAS Secrets
- Build a signed .ipa using the production provisioning profile
- Takes ~20-40 minutes on EAS build queue

Step 4 — Submit the completed build to TestFlight:
```bash
cd c:/VIIBE/VibeApp/frontend
eas submit --platform ios --latest
```
When prompted, select "App Store Connect" and confirm the bundle identifier com.viibe.app.

Step 5 — After submission completes, log the EAS build URL and the App Store Connect build ID in the summary. The build will be in "Processing" status in App Store Connect for ~10-30 minutes before appearing in TestFlight.

Step 6 — Once the build appears in TestFlight, create an External Testing group named "Beta Testers" and add at least one tester email. Copy the TestFlight public link.

Note on app.json version: If the build fails with a version conflict, increment the build number in app.json `"version"` field or use the EAS remote autoIncrement (already configured in production profile).
